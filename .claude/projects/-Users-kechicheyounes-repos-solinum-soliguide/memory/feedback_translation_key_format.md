Translation keys use UPPER_SNAKE_CASE format (e.g. `REGULARIZATION`, not `categories.regularization`).

**Why:** Project convention — confirmed by user when suggesting wrong format.

**How to apply:** Whenever referencing or suggesting translation keys for categories or any other entity, always use UPPER_SNAKE_CASE.
