## Notes d'utilisation

### Pour importer dans Notion

1. Copiez le tableau ci-dessus
2. Dans Notion, créez une page et collez le contenu
3. Notion détectera automatiquement le format markdown et créera une table de base de données

### Structure du tableau

- **Catégorie actuelle** : Nom de la catégorie actuellement en production
- **Date de création** : Date à laquelle la catégorie actuelle a été créée
- **Évolution SEO URL** : Chaîne de transformation des URLs pour les redirections (format: `ancien-url → url-intermediaire → url-actuelle`)
- **Historique des changements** : Chronologie complète des identifiants et labels

### Logique des SEO URLs

1. **URL Legacy** : L'URL du système numérique (ex: `sante`, `addiction`)
2. **URL Nommée** : Quand la catégorie passe en identifiant textuel, l'URL devient le nom en minuscule avec underscores (ex: `HEALTH` → `health`)
3. **Fusions** : Pour les catégories fusionnées, deux branches d'URLs sont tracées séparément jusqu'au point de fusion
4. **URLs inchangées** : Quand l'URL est restée identique malgré un changement d'identifiant interne

### Guide pour les redirections

**Redirections simples (1 niveau) :**

```
sante → health
douche → shower
```

**Redirections multi-niveaux (2+ niveaux) :**

```
vehicule → provision_of_vehicles → personal_vehicle_access
permis → mobility_assistance → mobility_financing
```

**Redirections avec fusion (2 branches) :**

```
Branche 1: covoiturage → carpooling → transportation_mobility
Branche 2: chauffeur → chauffeur_driven_transport → transportation_mobility
```

### Cas particuliers

**Fusions de catégories :**

- `TRANSPORTATION_MOBILITY` : Fusion de 2 branches (covoiturage et chauffeur) le 2025-12-31
- `FOOD_DISTRIBUTION` : Fusion de 2 branches (distribution-repas et restauration-assise) le 2025-12-31
