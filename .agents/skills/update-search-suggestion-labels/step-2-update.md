# Step 2 — Update the label in each file

## File location

```
packages/common/src/search-suggestions/data/{country}/{lang}.json
```

Each entry that matches the category looks like:

```json
{
  "label": "<old label>",
  "categoryId": "regularization",
  "slug": "regularization",
  "synonyms": [...],
  "type": "CATEGORY",
  "lang": "es",
  "country": "es",
  "seoTitle": "...",
  "seoDescription": "..."
}
```

## What to change

Only update the `"label"` field. Do **not** touch `synonyms`, `seoTitle`, or `seoDescription`.

## For each (country, lang) pair that has the entry

1. Read the file (required before editing)
2. Replace the `"label"` line with the new value from `translations/locales/{lang}.json`

Example edit:
```
old: "label": "Regularización",
new: "label": "Regularización extraordinaria",
```

## Verify

After all edits, confirm with:

```bash
for f in packages/common/src/search-suggestions/data/{country}/*.json; do
  lang=$(basename "$f" .json)
  result=$(grep -A1 '"categoryId": "{categoryId}"' "$f" | grep '"label"' || true)
  echo "$lang: $result"
done
```

All labels should now match their respective `CAT_*` translation keys.

## Important: do not update seoTitle / seoDescription

The SEO fields are generated by the Claude API (`categories:sync` script) and may not yet reflect the new category name. Leave them as-is — they will be refreshed in the next sync run.
